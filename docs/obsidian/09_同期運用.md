# 09_同期運用

## 方針

Obsidian側を日々の設計作業面とし、repo側の `docs/obsidian/` に同じMarkdownを保存する。

repoにpushする前に同期チェックを行う。

## パス

- Obsidian: optional external workspace via `RETAIL_AI_OPS_OBSIDIAN_DIR`
- Repo mirror: `docs/obsidian`

## コマンド

Obsidianからrepoへ同期:

```bash
python3 tools/sync_obsidian_docs.py --direction obsidian-to-repo
```

repoからObsidianへ同期:

```bash
python3 tools/sync_obsidian_docs.py --direction repo-to-obsidian
```

差分確認:

```bash
python3 tools/sync_obsidian_docs.py --direction check
```

## 運用ルール

- 設計書を更新したら同期チェックを通す
- 片側だけに存在するファイルを放置しない
- `.DS_Store` は同期対象外
- 外部組織や製品に関する事実主張は公開URLを残す
- 仮説は仮説として書く
- repo内の実装が進んだら、設計書の受入基準とtrace/eval仕様も更新する
